# QuizSpark Design System Implementation Plan

## Overview

Transform the existing Quizaro frontend from its current white-minimalist/blue-accent design to a **playful, competitive, and gamified** design system featuring bold yellow energy, electric purple accents, and refreshing cyan. The new design channels game show excitement and leveling-up satisfaction into a cohesive visual language for quiz/test preparation.

---

- Add `Righteous`, `Inter`, and `Space Mono` to `app/layout.tsx` via `next/font/google`

## Phase 2: Component Library Updates

### 2.1 Buttons
Create/Update button variants across the codebase:
- **Primary**: `bg-primary text-zinc-900 hover:bg-primaryHover`
- **Secondary**: `bg-white text-secondary border-2 border-secondary hover:bg-secondaryHover`
- **Ghost**: `bg-transparent text-secondary hover:bg-secondaryHover`
- **Destructive**: `bg-error text-white hover:bg-errorHover`
- **Sizes**: Small, Medium, Large with specific padding/font-size combos
- **Disabled**: `opacity-40 cursor-not-allowed grayscale`

**Files affected:**
- All action buttons in dashboards, quiz interface, modals, headers

### 2.2 Cards
Update all card components:
- **Base**: `bg-[#FAFAFA] border-2 border-[#E4E4E7] rounded-md p-6 shadow-md hover:shadow-lg transition-all duration-200`

- **Score Card**: Centered metric + rank display

**Files affected:**
- `components/AdminTestCard.tsx`
- `components/AdminFolderCard.tsx`
- `components/FeatureCard.tsx`
- Dashboard card sections

### 2.3 Inputs
Update form inputs:

**Files affected:**

### 2.4 Chips
Create new chip components:

**New files to create:**
- `components/FilterChip.tsx`

### 2.5 Checkboxes & Radio Buttons
Update to new design specs:
- **Checkbox**: 22px, 6px corners, primary yellow when checked

## Phase 5: Animation & Interaction Polish

- **Correct Answer**: Green glow (`shadow-correct`) + subtle scale animation, within 200ms
- **Wrong Answer**: Red shake (`shadow-wrong`) + gentle nudge, show correct answer with explanation
- **Streak Milestone**: Flame animation + badge pulse on 3/5/10/20 streaks
- **Level Up**: Modal with star animation + confetti burst

### 5.2 Timer Gradient
- Implement smooth color transition: `#06B6D4` (plenty) → `#F59E0B` (warning) → `#EF4444` (critical)
- Remove abrupt alarm states

### 5.3 Micro-interactions
- Add hover lift to all cards (`shadow-md` → `shadow-lg`)
- Add focus rings to all interactive elements (`shadow-focus`)
- Add smooth transitions (200ms ease) to all state changes
- Add celebration animations for achievements

## Phase 6: Accessibility & Quality

### 6.1 Colorblind Support
- Add icons (✓/✗) to all correct/incorrect feedback states
- Ensure sufficient contrast ratios for all text/background combinations
- Test all interactive states with reduced motion preference

### 6.2 Typography Hierarchy Audit
- Verify all headings use Righteous with correct sizes
- Verify all body text uses Inter with correct weights
- Verify all monospace elements (scores, timers) use Space Mono

### 6.3 Spacing Consistency
- Replace arbitrary spacing values with design system tokens (sp-1 through sp-8)
- Ensure 8px grid alignment throughout

### 6.4 Remove Dead Code
- Delete unused components: `FeatureCard.tsx`, `AdminNavbar.tsx`, `ThemeToggle.tsx` (if not needed)

## Implementation Order

1. **Foundation** (Phase 1) → Config, fonts, colors, shadows
2. **Components** (Phase 2) → Buttons, cards, inputs, chips, form elements
3. **Core Pages** (Phase 3) → Landing, auth, dashboards, quiz, results
4. **Global UI** (Phase 4) → Nav, modals, tooltips, help, loading
5. **Polish** (Phase 5) → Animations, feedback, micro-interactions
6. **Quality** (Phase 6) → Accessibility, typography audit, spacing, cleanup

---

## Risk & Considerations

- **High Risk**: Quiz interface redesign — must preserve anti-cheat functionality, timer logic, and submission flow
- **Medium Risk**: Dashboard layouts — heavy restructuring of card grids and data tables
- **Low Risk**: Landing/auth pages — mostly cosmetic updates
- **Dependency**: Google Fonts must load properly for typography to render correctly
- **Testing**: Need to verify all interactive states (hover, focus, disabled, error) after color changes
- **Performance**: Confetti/celebration animations should use lightweight libraries or CSS-only solutions
